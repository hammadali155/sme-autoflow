# SME AutoFlow: FastAPI Backend Documentation

This document describes the technical architecture, endpoints, and the complete execution workflow of the FastAPI backend in the **SME AutoFlow** project.

## Overview

The backend is built with **FastAPI** (`api/main.py`) and serves as the core intelligence hub for the application. It receives natural language requests from the frontend, orchestrates the entire machine learning and retrieval-augmented generation (RAG) pipeline, and ultimately returns a fully deployable n8n workflow JSON.

**Key Technologies Used:**
- **FastAPI / Uvicorn:** For high-performance, asynchronous REST API delivery.
- **Pydantic:** For data validation and strictly typed request/response schemas.
- **Joblib / Scikit-Learn:** For loading and executing trained ML models (`intent_classifier`, `node_recommender`).
- **ChromaDB / Sentence-Transformers (via RAG module):** For semantic search of workflow templates.
- **Google GenAI SDK (via Generator module):** For prompting Gemini 2.0 Flash to stitch the final JSON.

---

## Technical Details

### 1. App Lifespan and Model Loading
FastAPI handles model loading through an asynchronous `@asynccontextmanager` (`lifespan()`).
When the server starts:
- The `intent_pipeline` (TF-IDF + classification model) and its `label_encoder` are loaded into application state (`app.state`).
- The `node_pipeline` (TF-IDF + multi-label classification) and its `MultiLabelBinarizer` are similarly loaded.
- This ensures models are loaded into memory exactly once at startup, preventing latency on individual requests.

### 2. Pydantic Schemas
Strict typing ensures robust API interactions:
- **`WorkflowRequest`**: Requires a `prompt` (min 10, max 5000 chars) and an optional `top_k` (from 1 to 10) representing the number of templates to semantically retrieve.
- **`PredictionResponse`**: Returns the intermediate machine learning results (`intent`, `intent_confidence`, `predicted_nodes`, `similar_templates`).
- **`WorkflowResponse`**: Extensive output containing all machine learning results *plus* the `workflow_json` generated by Gemini.

### 3. API Endpoints

| Method | Endpoint | Tags | Description |
| :--- | :--- | :--- | :--- |
| **GET** | `/health` | Health | Validates the API is live and confirms if the machine learning models have successfully been loaded into memory. |
| **POST** | `/predict` | ML | Executes steps 1-3 (Intent Classification, Node Recommendation, RAG). Returns predicted metadata without hitting the Gemini LLM. Useful for testing UI previews. |
| **POST** | `/generate` | Generator | The primary endpoint. Executes the complete sequence and returns the fully generated n8n workflow JSON. |

## Complete Execution Workflow (`/generate`)

When a user submits a natural-language intent via `POST /generate`, the FastAPI backend performs the following deterministic multi-stage workflow:

### Stage 1: Intent Classification
The string `prompt` is passed through the pre-loaded intent pipeline (`app.state.intent_pipeline`).
- The system returns an intent category (e.g., `"data_sync"`) and a confidence score.

### Stage 2: Node Recommendation
The prompt is queried against the multi-label Node Recommender (`app.state.node_pipeline`).
- The system predicts an array of required n8n nodes (e.g., `["gmail", "postgres", "if"]`).

### Stage 3: RAG Retrieval (ChromaDB)
The `retrieve()` function queries the local vector database.
- It calculates semantic similarity between the user's prompt and 1,000 embedded n8n templates.
- It returns the top `K` most relevant n8n template JSONs to act as structural scaffolding.

### Stage 4: Generative LLM Processing (Gemini)
The backend invokes `generate_workflow()` (from `generator/gemini_generator.py`). It feeds the following rich context into the Gemini 2.0 model:
1. The raw user `prompt`.
2. The ML-predicted `intent`.
3. The ML-predicted `nodes`.
4. The JSON bodies of the `similar_templates`.

*Gemini intelligently merges the user's specific context with the technical scaffolding extracted from the RAG templates.*

### Stage 5: Response Delivery
The FastAPI endpoint receives the parsed JSON workflow from the generator, validates it, and returns the final `WorkflowResponse` to the Streamlit UI, where the user can inspect or deploy it directly to an n8n instance.
